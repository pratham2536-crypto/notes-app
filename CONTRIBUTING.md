# Contributing to Journal App

First off, thank you for considering contributing to the Journal App! It's people like you that make open source such a fantastic community.

## How Can I Contribute?

### Reporting Bugs
If you find a bug, please create an issue on GitHub. Include:
- Your operating system and browser version.
- Steps to reproduce the bug.
- Expected vs actual behavior.

### Suggesting Enhancements
Have a cool idea for a feature? Create an issue with the tag `enhancement`. Describe the feature in detail and explain why it would be useful for the community.

### Submitting Pull Requests
1. Fork the repository and create your branch from `main`.
2. Make your changes in your branch.
3. If you've changed styling, ensure it follows our existing variables in `style.css`.
4. Ensure your code is clean and well-commented.
5. Create a Pull Request against the `main` branch.

## Code Style Guide
- **HTML**: Use semantic HTML5 tags where possible.
- **CSS**: Use the existing custom properties (CSS variables) for colors, spacing, and typography.
- **JavaScript**: Use modern ES6+ features (arrow functions, const/let, template literals). Keep logic modular.

We look forward to reviewing your contributions!
